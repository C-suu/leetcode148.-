### （1）题目白话文解释

题目的核心诉求是：给定一个内部节点顺序被打乱的**链表**，要求将其中所有节点按照数字从小到大的顺序重新排列，最后返回排好序的链表的第一个节点（头节点）。

**生僻/关键概念解释：**

  * **链表 (Linked List)**：一种数据结构。不同于数组在内存中是挨在一起的连续存放，链表的节点在内存中是散落的。每个节点包含两部分：一个是自己存的数据（`val`），另一个是指向下一个节点的指针/地址（`next`）。就像寻宝游戏里的线索卡片，必须拿到当前卡片，才能知道下一张卡片藏在哪。
  * **升序排列**：即从小到大排列。
  * **时间复杂度限制（进阶要求）**：题目进阶要求时间复杂度为 $O(n \log n)$，在排序算法中，通常只有快速排序、归并排序、堆排序等能达到这个效率。对于链表而言，归并排序是最经典且稳定的解法。

### （2）代码解题思路来源

这段代码采用的是**归并排序（Merge Sort）算法，核心思想是“分而治之”**。

解题思路分为两大阶段：“分（拆解）”和“治（合并）”：

1.  **“分”——寻找中点并切断**：
      * 面对一个长链表，排序是很复杂的。但如果链表只有一个节点，那它天然就是有序的。因此，第一步就是不断把链表从中间一分为二。
      * **快慢指针法**：为了找到链表的中间节点，安排两个指针同时从头出发。慢指针每次走1步，快指针每次走2步。当快指针走到链表尽头时，慢指针刚好停在链表的中点。
      * 找到中点后，将链表从这里无情切断（将中点前一个节点的 `next` 设置为空），变成左右两个相对较短的链表。
2.  **“治”——合并有序链表**：
      * 通过递归，一直切分，直到所有子链表都只剩1个节点（天然有序）。
      * 接下来开始“合并”。准备一个虚拟的“哑节点（Dummy Node）”作为新链表的开头。
      * 同时比较左右两个有序链表的头节点，谁的值更小，就把谁接到哑节点后面。随后该链表的指针后移一位，继续比较，直到两段链表就像拉链一样严丝合缝地交错合并成一个更长的有序链表。

### （3）带有注释的代码

```python
class Solution:
    def sortList(self, head: ListNode) -> ListNode:
        # 递归终止条件：如果链表为空，或者只有一个节点，直接返回，无需排序
        if not head or not head.next: 
            return head 
        # 阶段一：使用快慢双指针寻找链表的中点，准备“分”
        # 初始时，慢指针在起点，快指针在第二个节点
        slow, fast = head, head.next
        # 当快指针没有越界时，继续遍历
        while fast and fast.next:
            # 快指针每次走两步，慢指针每次走一步
            fast, slow = fast.next.next, slow.next
        # 此时 slow 停在中点。保存右半段的头节点为 mid，然后将 left 段的尾巴切断
        mid, slow.next = slow.next, None 
        # 对切断后的左半段和右半段分别进行递归排序
        left, right = self.sortList(head), self.sortList(mid)
        # 阶段二：“治”，合并两个已经有序的左右链表
        # 创建一个值为 0 的哑节点（辅助节点），h 用来在后面不断延伸，res 用来记住起始位置
        h = res = ListNode(0)
        # 当左右两个链表都还有剩余节点时进行比较
        while left and right:
            # 如果左边节点的值较小，将左节点接入新链表，并向后移动左指针
            if left.val < right.val: 
                h.next, left = left, left.next
            # 否则，将右节点接入新链表，并向后移动右指针
            else: 
                h.next, right = right, right.next
            # 将合并指针 h 也向后移动一位，准备挂载下一个节点
            h = h.next
        # 循环结束后，肯定有一个链表先被合并完。将另一个未合并完的链表整串直接接在末尾
        h.next = left if left else right
        # 返回哑节点的下一个节点，也就是真正合并后有序链表的头节点
        return res.next
```

### （4）每一行代码详细解释

1.  `class Solution:`：定义执行算法的类。
2.  `def sortList(self, head: ListNode) -> ListNode:`：定义核心函数，接收链表头节点 `head`，返回排序后的头节点。
3.  `if not head or not head.next: return head`：防御性编程及递归边界条件。若链表为空（`not head`）或长为1（`not head.next`），这本身即是有序状态，直接返回原节点终止递归。
4.  `slow, fast = head, head.next`：初始化快慢指针。注意此处 `fast` 比常规设在 `head` 提前了一步，这是为了保证在链表长度为偶数时，`slow` 指针刚好停在**左半部分的末尾**，从而方便精准切断。
5.  `while fast and fast.next:`：启动寻找中点的循环。由于快指针一次跨两步，必须保证它当前所在节点和下一个节点都不是空，否则会触发空指针异常。
```python
while fast and fast.next:
这个不理解，比如链表1 2 3 4，fast是2，fast.next是3，那fast.next.next是4，fast.next.next是空就不影响吗？

`fast.next.next` 是空（`None`）完全不影响，也不会报错。例子 `1 -> 2 -> 3 -> 4` 。
在 Python 中，把 `None` 赋值给一个变量是完全合法的。
代码中的赋值语句是：`fast = fast.next.next`
只要 `fast` 存在，且 `fast.next` 存在，那么 `fast.next.next` 顶多就是一个 `None`。
这句代码仅仅是**把 `fast` 指针指向了 `None`**，这并没有任何危险。
真正的危险在于**下一轮循环开始时**。如果 `fast` 已经是 `None` 了，程序再去查问 `fast.next`，就会立刻崩溃。
这就是 `while fast and fast.next:` 存在的意义：
是一个保镖，负责在进入循环体执行 `fast.next.next` 之前，提前拦截所有危险。
```
6.  `fast, slow = fast.next.next, slow.next`：快指针跨越2个节点，慢指针跟进1个节点。
7.  `mid, slow.next = slow.next, None`：这是极其关键的断链操作。此时 `slow` 是左半段的最后一个节点。第一步将 `slow` 后面的节点（即右半段的起点）赋值给暂存变量 `mid`；第二步将 `slow.next` 强行设为 `None`，这就在物理上将一条长链表扯断成了前后独立的两条。
8.  `left, right = self.sortList(head), self.sortList(mid)`：进入深度递归。将断开后的前半段（头仍是 `head`）和后半段（头是 `mid`）分别丢入 `sortList` 函数中继续执行拆分排序，最终会返回两条分别排好序的链表头节点，分别命名为 `left` 和 `right`。
```python
`left, right = self.sortList(head), self.sortList(mid)` 
它不是同时执行的，而是先一头扎进左边（深度优先），等左边彻底排好序返回了，再一头扎进右边，最后把左右合并。
以链表 `4 -> 2 -> 1 -> 3` 为例：
### 第一层（顶层）：处理完整链表 `[4, 2, 1, 3]`
1.找中点并切断：`slow` 停在 `2`。链表被切断为左半段 `head = [4, 2]`，右半段 `mid = [1, 3]`。
2.准备执行等式：遇到了 `left, right = self.sortList([4, 2]), self.sortList([1, 3])`。
3.暂停：计算机必须先算出等号右边第一个函数 `sortList([4, 2])` 的结果，才能继续。
于是，带着 `[4, 2]` 进入下一层（左分支）。

### 第二层（左分支）：处理局部链表 `[4, 2]`
1.  **找中点并切断**：`slow` 停在 `4`。链表被切断为左半段 `head = [4]`，右半段 `mid = [2]`。
2.  **准备执行等式**：遇到了 `left, right = self.sortList([4]), self.sortList([2])`。
3.  **程序再次暂停**：必须先算 `sortList([4])`。带着 `[4]` 进入第三层（左左分支）。

#### 第三层（左左分支）：处理单节点 `[4]`
1.  触发递归终止条件：`if not head or not head.next:`。因为链表只剩一个节点 `4`，天然有序。
2.  **直接返回**：返回节点 `[4]`。（**“归”的开始**）
*(退回到第二层)*
* 第二层此时拿到了结果：`left = [4]`。接着执行等式的下一半：去算 `sortList([2])`。

#### 第三层（左右分支）：处理单节点 `[2]`
1.  同样触发终止条件，只有一个节点。
2.  **直接返回**：返回节点 `[2]`。

*(再次退回到第二层)*
* 第二层的等式终于计算完毕：`left = [4]`，`right = [2]`。
执行合并（治）：第二层开始执行后半段的 `while left and right:` 合并逻辑。
将 `[4]` 和 `[2]` 穿针引线，合并成有序链表 `[2, 4]`。
向上返回：第二层任务圆满完成，将合并好的 `[2, 4]` 返回给第一层。（**至此，顶层等式的左半边终于算完了**）

### 重新回到第一层（顶层）
* 第一层拿到左边返回的结果：**`left = [2, 4]`**。
* 第一层终于可以松一口气，开始执行等式的右半边：`self.sortList([1, 3])`。
带着 `[1, 3]` 进入第二层（右分支）。

### 第二层（右分支）：处理局部链表 `[1, 3]`
*(执行过程与左分支完全对称)*
1.  切断为 `[1]` 和 `[3]`。
2.  进入第三层算 `sortList([1])`，直接返回 `[1]`。
3.  进入第三层算 `sortList([3])`，直接返回 `[3]`。
4.  在本层得到 `left = [1]`，`right = [3]`。执行合并逻辑，合并为 `[1, 3]`。
5.  **向上返回**：将有序的 `[1, 3]` 返回给第一层。

### 最终大结局：回到第一层（顶层）
* 顶层等式彻底执行完毕，成功拿到了两个分别排好序的半成品：
    * **`left = [2, 4]`**
    * **`right = [1, 3]`**
* **终极合并**：顶层执行它自己的 `while left and right:`，
像拉拉链一样将 `[2, 4]` 和 `[1, 3]` 进行比对合并。
* 合并完成：`[1, 2, 3, 4]`。
```
10.  `h = res = ListNode(0)`：创建值为空意义的哑节点（Dummy Node），这是处理链表合并时的常规套路。`res` 作为固定的锚点记住开头，`h` 作为穿针引线的移动游标。
11. `while left and right:`：当左右两条子链表都未被完全遍历完时，持续进行大小比对。
12. `if left.val < right.val: h.next, left = left, left.next`：若左边链表当前节点值更小。首先令游标 `h` 的下一个节点指向这个较小的 `left` 节点；随后将 `left` 指针自身向后移动一位，准备下一轮比对。
13. `else: h.next, right = right, right.next`：反之，若右边节点更小或相等，则将 `right` 节点挂载到 `h` 后面，并将 `right` 指针后移。
14. `h = h.next`：无论挂载了左边还是右边，新链表的游标 `h` 都必须向前步进一位，停在刚刚挂载好的节点上，等待接收下一个。
15. `h.next = left if left else right`：当循环打破，意味着必然有一条链表已经全部挂载完毕。此时利用 Python 三元表达式，直接将另一条还没空（剩余全为较大的数）的链表，整串挂载到新链表游标 `h` 的尾部。
16. `return res.next`：因为 `res` 是当初那个虚拟的哑节点，其挂载的下一个节点 `res.next` 才是拥有真实数据的有序链表新头部，将其返回。
